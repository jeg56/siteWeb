Cycle en V 

Le concept

Le modèle du cycle en V (en comparaison avec les méthodes dites agiles) est un modèle conceptuel de gestion de projet imaginé afin de limiter un retour aux étapes précédentes en cas d'anomalie

Les phases de la partie montante doivent renvoyer de l'information sur les phases en vis-à-vis lorsque des défauts sont détectés, afin d'améliorer le logiciel.
Ce modèle met en évidence la relation entre les phases du début de cycle et celles de fin.


Gestion de projet

les étapes :​
spécifications : phase fonctionnelle - 25 % du temps du projet
codage & tests unitaires : phase technique - 50 % du temps du projet
tests de validation : phase fonctionnelle - 25 % du temps du projet

Documentation d’un projet :​
Phase de cadrage : Phase durant laquelle vous allez vous entourer d’experts pour bien délimiter le projet .À se rendre compte de tous les besoins et de toutes les contraintes.Une note de cadrage doit préciser :
1. Les enjeux du projet
2. Les utilisateurs du produit (ou de l’évolution)
3. Les besoins à couvrir par le produit pour ces utilisateurs (vision du produit)
4. Les contraintes à respecter sur le SI (urbanisme fonctionnel, architecture technique)
5. Le budget alloué au projet

Phase de spécification
La spécification d'un projet est un ensemble explicite d'exigences à satisfaire par un service.
La phase de spécification permet de décrire le besoin fonctionnel (également technique) dans le but d’élaborer un projet. La priorité est de favoriser la lisibilité et de rendre l’information facile à trouver pour vos lecteurs. Privilégiz donc les schémas logiques pour présenter les différents parcours d’exploitation de l’outils.
Un dossier de spécification doit préciser :
1. Quelle est la mission demandée ?
2. Quel est le périmètre global ?
3. Quels sont les objectifs du projet et quels sont les facteurs clé de son succès s'ils sont définis ?
4. Les règles de gestion & caractéristiques (Les contraintes d’accessibilité : (ex: norme RGAA ),responsive design, Performance globale ...)
5. Un rappel du planning : les principaux jalons et les livrables.

Il est souvent nécessaire d'intégrer à son projet des applications déjà existantes. Il faut les lister dans cette partie. Présentez leur rôle et donnez un lien vers leur documentation.

Phase de Recette
Phase Définition
Tests unitaires En général pris en charge par le développeur sur son poste de travail et avec les outils adaptés à la technologie de développement.Ils vérifient les lignes de code d’un programme élémentaire.
Tests d’intégration En général pris en charge par le développeur, en environnement d’intégration. ils vérifient que les programmes élémentaires s’assemblent entre eux et que les interfaces internes fonctionnent.Peut nécessiter la mise en œuvre de bouchons ou de simulateurs
Recette usine En général pris en charge par les fonctionnels, en environnement de Qualification Ils vérifient que les applications communiquent entre elles (en interne, comme en externe), sans bouchon. C’est une étape de vérifications qui permet de livrer au métier un projet conforme et opérant dans le SI complet.
