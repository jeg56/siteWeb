Principe du MapReduce
MapReduce consiste à découper les données en sous-ensembles de plus petite taille, que nous appelons des lots et à affecter chaque lot à une machine du cluster permettant ainsi leur traitement en parallèle. Il suffira ensuite d'agréger l'ensemble des résultats intermédiaires obtenus pour chaque lot pour construire le résultat final.

4 étapes distinctes :
- Découper (SPLIT)
- Transformer (MAP)
- Grouper (SHUFFLE)
- Réduire (REDUCE)


Le Découpage

L'ensemble des données à traiter est découpé en plusieurs lots ou sous-ensembles.

L'opération map (Transformer)

doit retourner une liste de paires(clé, valeur).

SHUFFLE and SORT ( gouper et trier)

Avant d'être envoyés à l'étape REDUCE, les résultats intermédiaires sont regroupés et triés par clé.
Cette étape est entièrement gérée par le framework d'exécution de MapReduce de manière distribuée.

L'étape REDUCE

Cette étape consiste à appliquer l'opération reduce, spécifiée pour notre problème, à chaque clé en parallèle. Elle agrège tous les résultats intermédiaires associés à une même clé et renvoie donc pour chaque clé une valeur unique.


• Le rôle des developpeurs d'applications distribuées, c'est donc de penser en MapReduce :
1. Choisir une manière de découper les données afin que l'opération MAP soit parallélisable.
2. Choisir la clé à utiliser pour le problème ciblé.
3. Écrire le code de la fonction pour l'opération MAP.
4. Écrire le code de la fonction pour l'opération REDUCE.
